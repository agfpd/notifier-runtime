# 04 — Watcher

**Русский** · [English](../04-watcher.md)

Пир `watcher` запускает долгоживущий скрипт и превращает его вывод в сигналы. Каждая непустая строка, которую скрипт печатает в stdout, становится одним сообщением получателю — строка и есть полезная нагрузка. Агент-пир регистрирует его вызовом `send_to_peer` со `script` и `target`.

```
send_to_peer(watcher, {"script": "tail -F /var/log/app.log | grep ERROR", "target": "self"})
```

Каждая совпавшая строка приходит на `self` отдельным сигналом. Watcher — правильный инструмент, когда «что-то произошло» не может ждать опроса: совпадение в логе, появившийся файл, превышенный порог.

## Как вывод становится сигналами

Скрипт запускается через shell, поэтому полноценный конвейер (`tail -F x | grep y`) допустим. Watcher читает stdout построчно:

- **Каждая непустая строка форвардится дословно** как отдельный сигнал получателю. Завершающие переводы строки и `\r` (CRLF) срезаются; пустые строки отбрасываются.
- **Частичные строки буферизуются**, пока скрипт не допечатает остаток. Финальная строка без завершающего перевода строки сбрасывается при выходе скрипта.
- **stderr логируется, но никогда не форвардится.** Диагностика в stderr остаётся в логе рантайма и не доходит до получателя как сигнал.

Так что скрипт, желающий поднять N тревог, печатает N строк; желающий молчать — не печатает ничего.

## Перезапуски

Долгоживущий монитор не должен завершаться. Watcher считает **любой** выход сбоем — даже чистый код `0` — и перезапускает скрипт с экспоненциальным backoff, чтобы скрипт, умирающий мгновенно, не молотил хост:

```
1s → 2s → 4s → 8s → … с потолком 60s
```

Прогон длиной хотя бы в одну секунду или напечатавший хотя бы одну строку считается здоровым и сбрасывает backoff. Backoff накапливается только при по-настоящему быстрой смерти — перезапусках за доли секунды.

Сбой запуска (команды нет, она не исполняема) обрабатывается как любой другой сбой: backoff и повтор, с учётом в предохранителе от зацикливания ниже.

## Предохранитель от зацикливания на падениях

Безнадёжно сломанный скрипт — опечатка в первой строке, мгновенный выход — иначе перезапускался бы вечно. Предохранитель это останавливает: **5 сбоев за 60 секунд** взводят его. При срабатывании watcher прекращает перезапуски, шлёт тревогу **вам** (владельцу триггера) и громко логирует. В тревоге сказано, какой watcher сдался и почему, так что сломанный монитор всплывает, а не молотит в фоне.

## Детект зависания: `heartbeatSec`

Задайте `heartbeatSec`, чтобы ловить watcher, который жив, но застрял — процесс работает, но перестал выдавать вывод:

```
send_to_peer(watcher, {
  "script": "monitor-disk",
  "target": "ops",
  "heartbeatSec": 60
})
```

Если скрипт молчит дольше `heartbeatSec`, watcher считает это зависанием: убивает процесс (SIGKILL), шлёт тревогу владельцу и перезапускает его. Таймер heartbeat переармируется на каждой форварднутой строке, так что скрипт, продолжающий выдавать вывод, его не взводит. Без `heartbeatSec` молчание разрешено бессрочно — это правильно для watcher'а, который и должен молчать, пока что-нибудь не случится.

## Поля

| Поле | Обязательно | Значение |
|------|-------------|----------|
| `script` | да | долгоживущая команда; каждая непустая строка stdout форвардится |
| `target` | да | пир-получатель сигнала или `"self"` для пира-отправителя |
| `heartbeatSec` | нет | максимум молчания до перезапуска + тревоги (положительное число) |
| `id` | нет | стабильное имя; опустите — выведется из содержимого |
| `topic` | нет | короткая метка на каждом сигнале (≤200 символов) |
| `fallback` | нет | пир(ы) эскалации, если получатель недостижим — см. [06 — Эскалация](06-эскалация.md) |

Как и сообщения таймера, сигнал watcher'а попадает в **новую сессию** у получателя. Держите каждую строку самодостаточной — она должна быть понятна сама по себе, ведь приходит без окружающего контекста.

## Правка и удаление

Зарегистрируйте заново с тем же `id`, чтобы заменить watcher на месте; рантайм перезапускает скрипт, только если реально изменились `script`, `target`, `heartbeatSec`, `topic` или `fallback`. Неизменные watcher'ы продолжают работать через перезагрузку. Список и удаление:

```
send_to_peer(watcher, {"cmd": "list"})
send_to_peer(watcher, {"cmd": "unregister", "id": "error-watch"})
```

При остановке рантайм чисто убивает все процессы watcher'ов и не перезапускает их. Полный формат сообщения — в [05 — Регистрация триггеров](05-регистрация-триггеров.md).
