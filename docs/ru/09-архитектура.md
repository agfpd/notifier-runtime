# 09 — Архитектура

**Русский** · [English](../09-architecture.md)

Эта страница объясняет, как notifier-runtime подключается к iapeer: как он устанавливается, как iapeer провижнит и запускает его пиров, как идентичность и манифест связывают всё воедино. Для регистрации триггеров это знать не нужно — читайте, когда разворачиваете или отлаживаете рантайм.

## Один рантайм, две личности

Пакет — это один рантайм с именем `notifier`, несущий двух пиров. Один и тот же бинарь работает как любой из них; на старте он диспатчит по разрешённой личности:

- `timer` → планировщик (примитив TIME) → идентичность `notifier-timer`
- `watcher` → супервизор (примитив EVENT) → идентичность `notifier-watcher`

Личность пира разрешается из `PEER_PERSONALITY`, затем из локального профиля пира, затем по умолчанию в `timer`. Логика планирования и разбора не зависит от имени — имена использует только отображаемая идентичность.

Это зеркалит `telegram-runtime`: подключаемый рантайм, который iapeer провижнит и запускает, отдельный от собственного ядра iapeer.

## Шов установки

Установка — рукопожатие между пакетом (владеет своим деплоем) и фундаментом iapeer (владеет провижном).

```text
  npx -y @agfpd/notifier-runtime          iapeer install-runtime notifier
            │                                        │
            ▼                                        ▼
   self-install:                          читает манифест, затем для
   • бинарь лаунчера на PATH              каждого объявленного пира:
   • манифест записан в                   • запускает хук self-config
     <IAPEER_ROOT>/runtimes/              • пишет launchd-джоб
       notifier/runtime.json              • бутстрапит сессию
```

Пакет никогда не трогает реестр пиров или launchd напрямую — это домен iapeer. Он лишь объявляет, что он есть, в манифесте, и даёт iapeer действовать по нему.

## Манифест

`self-install` пишет `<IAPEER_ROOT>/runtimes/notifier/runtime.json`. Он объявляет рантайм для iapeer:

- **`runtime`** — `"notifier"`.
- **`version`** — штампуется из версии пакета.
- **`selfConfig`** — как вызвать пер-пировый хук конфигурации: `{command: <абсолютный путь к bin>, args: ["self-config"]}`. Абсолютный путь означает, что хук резолвится без зависимости от PATH.
- **`peers`** — объявленный набор: `timer` и `watcher`, оба `intelligence: "absent"` (они программные — ни модели, ни человека за ними), каждый с коротким описанием для реестра.

Манифест пишется атомарно и байт-в-байт идентичен при повторной установке. iapeer читает его во время `install-runtime`, чтобы знать, каких пиров провижнить и как их настроить.

## Провижн и запуск

Когда вы запускаете `iapeer install-runtime notifier`, iapeer:

1. Читает манифест.
2. Для `timer` и `watcher` запускает хук `self-config` (он пишет регистрационную самодокументацию каждого пира в его локальный профиль, сохраняя идентичность, выставленную фундаментом).
3. Пишет launchd-джоб на каждого пира (`com.iapeer.timer.plist`, `com.iapeer.watcher.plist`), прописывая путь лаунчера в джоб, чтобы он резолвился независимо от `PATH`.
4. Бутстрапит сессии.

С этого момента launchd держит двух пиров запущенными. Каждый выполняет `notifier-runtime run` под своей идентичностью, которую iapeer поднимает командой `iapeer run-infra <peer> notifier`.

## Состояние на диске

| Путь | Содержимое |
|------|------------|
| `<IAPEER_ROOT>/runtimes/notifier/runtime.json` | манифест рантайма (его читает iapeer) |
| `<peer-cwd>/.iapeer/peer-profile.json` | профиль каждого пира, включая его `notifier.triggers[]` |
| `~/.iapeer/peers-profiles.json` | реестр iapeer, который рантайм читает для поиска профилей пиров |
| `<IAPEER_ROOT>/state/notifier/<personality>/escalation/` | пер-пировый журнал упреждающей записи для сигналов в полёте |

## Что кому принадлежит

- **Пакет владеет:** self-install, манифестом, хуком `self-config`, разбором и хранением триггеров, планированием, супервизией и эскалацией.
- **iapeer владеет:** реестром пиров, launchd-джобами, последовательностью загрузки, провижном идентичности и транспортом `iapeer send`, через который рантайм доставляет.

Рантайм — не самостоятельный планировщик. Ему нужен iapeer, чтобы его провижнить, запустить и нести его сигналы.
