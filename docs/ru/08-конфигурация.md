# 08 — Конфигурация

**Русский** · [English](../08-configuration.md)

notifier-runtime почти не требует настройки. Одна переменная окружения предназначена для вас; остальные выставляет лаунчер iapeer или они зашиты как фиксированная политика. Эта страница перечисляет их все, чтобы ничего не оставалось загадкой.

## Что задаёте вы

### `NOTIFIER_FALLBACK_TARGET`

Глобальный backstop эскалации — конечный получатель, добавляемый в цепочку каждого сигнала после владельца триггера. Задайте его пиру, который должен ловить любой сигнал, для которого не удалось достать владельца, например командному дежурному пиру.

- **По умолчанию:** пусто. Без backstop владелец — терминальное звено, и свежей установке не нужна командная настройка.
- См. [06 — Эскалация](06-эскалация.md).

## Что задаёт iapeer

Эти приходят из окружения запуска, которое предоставляет iapeer; вручную вы их не задаёте, и менять их руками — обычно ошибка.

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `IAPEER_ROOT` | `$HOME/.iapeer` | корень состояния iapeer — определяет путь манифеста и журнала эскалации |
| `IAPEER_BIN_DIR` | `~/.local/bin` | куда `self-install` кладёт бинарь лаунчера |
| `IAP_BIN` | `iapeer` | команда доставки сигналов (`<IAP_BIN> send …`) |
| `NOTIFIER_RUNTIME_BIN` | — | абсолютный путь к лаунчеру, который iapeer прописывает в launchd-джоб, чтобы рантайм резолвился независимо от PATH |
| `IAPEER_PEER_PERSONALITY` | — | роль настраиваемого пира, читается командой `self-config` |
| `PEER_PERSONALITY` | `timer` | личность, которую принимает run-loop; откатывается к локальному профилю пира, затем к `timer` |

`IAPEER_ROOT` переопределяет корень состояния везде — и путь манифеста (`<IAPEER_ROOT>/runtimes/notifier/runtime.json`), и журнал (`<IAPEER_ROOT>/state/notifier/<personality>/escalation/`) выводятся из него.

## Фиксированная политика

Эти значения зашиты в рантайм и не настраиваются. Перечислены, чтобы вы знали поведение, а не потому что их можно менять.

### Timer

| Значение | Параметр |
|----------|----------|
| Таймаут гейта `check` | 30 секунд — зависшая проверка пропускается, fail-safe |
| Потолок сна run-loop | 60 секунд — периодическая переоценка, подхватывающая перезагрузки и переживающая скачок часов |
| Горизонт поиска cron | 4 года — выражение без срабатывания в нём (например 30 февраля) отклоняется как невозможное |

### Watcher

| Значение | Параметр |
|----------|----------|
| Backoff перезапуска | старт 1 секунда, удвоение, потолок 60 секунд |
| Порог здорового прогона | 1 секунда — прогон не короче сбрасывает серию backoff/сбоев |
| Предохранитель от зацикливания | 5 сбоев за 60 секунд взводят его; перезапуски прекращаются, владельцу шлётся тревога |

### Эскалация

| Значение | Параметр |
|----------|----------|
| Попыток на звено цепочки | 2 (одна попытка + один повтор) |
| Задержка между попытками на звене | 30 секунд |
| Жёсткий таймаут одной попытки доставки | 120 секунд |
| Лимит длины topic | 200 символов |

Что эти значения делают в контексте — см. [03 — Timer](03-timer.md), [04 — Watcher](04-watcher.md) и [06 — Эскалация](06-эскалация.md).
